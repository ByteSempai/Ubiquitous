FluorineFx.Hosting.HostManager

The HostManager object sits on top of all running FluorineFx application AppDomains and can be used to manage the applications.
Each application is hosted in its own Application Domain.

Setting up the runtime environment

The runtime uses the "applications" subdirectory as its base directory.
However it is possible to use the StartApplication method with any physical path (the directory name itself will act like a virtual directory on a Web server).

HostManager summary

HostManager.Start

The Start method will initialize the host.

HostManager.Stop

The Stop method will shutdown all the running applications.

HostManager.StartApplication

Starts a new application from the specified physical path (full path to the application folder)

HostManager.StopApplication

Stops an application identified by its Application Id.

Note: use GetRunningApplication to get the identity of the application.
Note: the runtime will raise an ApplicationShutdown event on a different thread after the application was stopped.

HostManager.GetRunningApplications

Returns an array of ApplicationInfo objects for each running application.

HostManager.GetRunningApplication

Returns an ApplicationInfo object identifying the application started from the specified physical path.

HostManager.ApplicationRestart

Event raised when an application was restarted.

HostManager.ApplicationShutdown

Event raised when an application was stopped.

HostManager.ApplicationError

Event raised when an application error occurred.
